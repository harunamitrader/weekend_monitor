# 今後の実装メモ

## 72時間チャート機能

将来的に、蓄積した 15 分ごとの価格履歴を使って、各銘柄の過去 72 時間分のチャートを表示できるようにしたい。

### 想定仕様

- 表示対象は直近 72 時間
- 15 分ごとの取得データを利用する
- 1 銘柄あたり最大でおよそ 288 点を描画する
- チャート対象の主値は `currentPrice` を第一候補とする

### 現在の土台

- `data/history/YYYY-MM-DD.json` に 15 分ごとの履歴が保存される
- `data/history/index.json` に履歴ファイル一覧が保存される
- この履歴を利用すれば、3 日分の価格推移データを組み立てられる

### 実装時の方針候補

#### 案1: 事前生成

- GitHub Actions 実行時に、直近 72 時間ぶんだけを整形した JSON を別途生成する
- 例: `docs/data/charts/<market-id>-72h.json`
- フロントはその軽量 JSON だけを読んで描画する

利点:

- ブラウザ側の処理が軽い
- Pages 公開に向く
- 履歴ファイルが増えても UI 側の負荷を抑えやすい

#### 案2: フロントで履歴結合

- ブラウザ側で `data/history/*.json` を読み、直近 72 時間ぶんを抽出する

利点:

- 生成ファイルの種類を増やさずに済む

注意点:

- 履歴が増えるほど読み込みが重くなる
- GitHub Pages の静的配信では事前生成より不利

### 推奨方針

実装時は、案1の「72時間ぶんの派生 JSON を事前生成する」方式を優先する。

### UIメモ

- 一覧カード内に小さなスパークラインを置く
- または、銘柄をクリックすると 72 時間チャートを展開する
- Swiss Design の方向性を崩さないよう、線・余白・ラベルは最小限にする

### データ処理メモ

- `stale: true` の点は補間せず、そのまま欠損扱いにする方が安全
- 全履歴を毎回読むのではなく、72 時間だけを切り出す
- 将来的に CSV ダウンロードにも転用できるよう、整形 JSON の構造はシンプルに保つ
